# Template de Prompt para Criar Rotas de API

Crie as rotas `GET /users/me` e `PUT /users/me`

## Descrição

Essas rotas gerenciam os dados de treino do usuário autenticado. O `GET` retorna os dados e o `PUT` cria ou atualiza (upsert).

## Requisitos Técnicos

- Use cases devem se chamar "GetUserTrainData" e "UpsertUserTrainData".
- Os dados de treino ficam diretamente no model `User` (campos `weightInGrams`, `heightInCentimeters`, `age`, `bodyFatPercentage`).
- As rotas pertencem ao módulo `users` existente em `src/users/`.

## Autenticação

- Rotas protegidas.

---

### Rota 1: `GET /users/me`

#### Request

```ts
interface Body {}
```

```ts
interface Params {}
```

```ts
interface Query {}
```

#### Response

```ts
interface StatusCode200 {
  userId: string
  userName: string
  weightInGrams: number | null
  heightInCentimeters: number | null
  age: number | null
  bodyFatPercentage: number | null // 1 representa 100%
}
```

#### Regras de Negócio

- A rota deve retornar os dados do usuário autenticado (via session).
- Se os dados de treino ainda não existirem (campos null), retornar os campos como `null` com status 200 (NÃO retornar 404).

---

### Rota 2: `PUT /users/me`

#### Request

```ts
interface Body {

  age: number
  bodyFatPercentage: number // 1 representa 100%
}
```

```ts
interface Params {}
```

```ts
interface Query {}
```

#### Response

```ts
interface StatusCode200 {
  userId: string
  userName: string
  weightInGrams: number
  heightInCentimeters: number
  age: number
  bodyFatPercentage: number
}
```

#### Regras de Negócio

- Criar ou atualizar os dados de treino do usuário autenticado.

## Padrão de Implementação

### DTO (`dto/users.dto.ts`)

- Zod schema para validação do body do PUT.
- Classes com `@ApiProperty` para documentação Swagger, padrão Input/Output por rota (ex: `GetUserTrainDataOutputDto`, `UpsertUserTrainDataInputDto`, `UpsertUserTrainDataOutputDto`).
- Tudo do mesmo model fica em um único arquivo.

### Repository (`repositories/users.repository.ts`)

- Criar `UsersRepository` com Injectable e `PrismaService`.
- Métodos para buscar e atualizar dados do usuário.

### Use Cases (`use-cases/`)

- `get-user-train-data.use-case.ts`: busca o usuário e retorna os dados.
- `upsert-user-train-data.use-case.ts`: atualiza os campos de treino do usuário.
- Injectable, recebem repositório via constructor.
- Sem try-catch.

### Controller (`users.controller.ts`)

- Atualizar o controller existente com as novas rotas.
- `@Get('me')` e `@Put('me')` no controller existente.
- Decorators de response do Swagger.
- Usa `ZodValidationPipe` para validação do body no PUT.
- O controller apenas delega para os use cases.

### Module (`users.module.ts`)

- Registrar repositório e use cases nos providers.
