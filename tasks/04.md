# Template de Prompt para Criar uma Rota de API

Crie a rota `GET /workout-plan`

## Descrição

Essa rota deve retornar o plano de treino ativo do usuário logado. Para os dias do plano de treino, NÃO retorne os `exercises`, mas apenas `exerciseCount`.

## Requisitos Técnicos

- Use case deve se chamar "GetWorkoutPlan".
- Reutilizar o `WorkoutPlanRepository` existente.

## Autenticação

- Rota protegida.
- O usuário precisa ser dono do plano de treino recebido.

interface Body {}

## Response

```ts
interface StatusCode200 {
  id: string
  name: string
  workoutDays: Array<{
    id: string
    weekDay: string
    name: string
    isRest: boolean
    coverImageUrl?: string
    estimatedDurationInSeconds: number
    exerciseCount: number
  }>
}
```

## Regras de Negócio

- Verificar se o plano de treino pertence ao usuário autenticado (`ForbiddenException`).
- Retornar `NotFoundException` caso o plano não exista.

## Padrão de Implementação

### DTO (`dto/workout-plan.dto.ts`)

- Classes com `@ApiProperty` servem como tipo e documentação Swagger.
- Seguir padrão Input/Output por rota (ex: `GetWorkoutPlanOutputDto`).
- Tudo do mesmo model fica em um único arquivo.

### Repository (`repositories/workout-plan.repository.ts`)

- Adicionar método no `WorkoutPlanRepository` para buscar o plano por ID com `_count` de exercises nos dias.

### Use Case (`use-cases/get-workout-plan.use-case.ts`)

- Injectable, recebe repositório via constructor.
- Valida existência do plano (`NotFoundException`) e ownership (`ForbiddenException`).
- Mapeia `_count.exercises` para `exerciseCount`.

### Controller (`workout_plan.controller.ts`)

- `@Get()` no controller existente.
- Decorators de response do Swagger.
- O controller apenas delega para o use case.

### Module (`workout_plan.module.ts`)

- Registrar o novo use case nos providers.
