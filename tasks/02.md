# Template de Prompt para Criar uma Rota de API

Crie a rota `PATCH /workout-plan/:planId/days/:dayId/sessions/:sessionId`

## Descrição

Essa rota completa uma sessão de treino específica, registrando a data/hora de conclusão.

## Requisitos Técnicos

- Use case deve se chamar "CompleteWorkoutSession".
- A rota pertence ao módulo `workout_session` existente em `src/workout_session/`.

## Autenticação

- Rota protegida.
- O usuário precisa ser dono do plano de treino recebido.

## Request

```ts
interface Body {
  completedAt: string // ISO String (2026-02-25T19:46:55.754Z)
}
```

```ts
interface Params {
  planId: string // uuid do WorkoutPlan
  dayId: string // uuid do WorkoutDay
  sessionId: string // uuid do WorkoutSession
}
```

```ts
interface Query {}
```

## Response

```ts
interface StatusCode200 {
  id: string
  completedAt: string // ISO String (2026-02-25T19:46:55.754Z)
  startedAt: string // ISO String (2026-02-25T18:30:00.000Z)
}
```

## Regras de Negócio

- Apenas o dono do workout plan pode completar a sessão de treino.
- A sessão deve existir e pertencer ao dia e plano informados.

## Padrão de Implementação

### DTO (`dto/workout-session.dto.ts`)

- Zod schema para validação do body (`completedAt`).
- Classes com `@ApiProperty` para documentação Swagger, padrão Input/Output por rota.
- Tudo do mesmo model fica em um único arquivo.

### Repository (`repositories/workout-session.repository.ts`)

- Adicionar/usar métodos no `WorkoutSessionRepository` existente.

### Use Case (`use-cases/complete-workout-session.use-case.ts`)

- Injectable, recebe repositórios via constructor.
- Valida ownership do plano (userId).
- Lança exceções do NestJS (`NotFoundException`, `ForbiddenException`).
- Sem try-catch.

### Controller (`workout_session.controller.ts`)

- `@Patch(':planId/days/:dayId/sessions/:sessionId')` no controller existente.
- Decorators de response do Swagger.
- Usa `ZodValidationPipe` para validação do body.
- O controller apenas delega para o use case.

### Module (`workout_session.module.ts`)

- Registrar o use case nos providers.
- Importa `WorkoutPlanModule` para acesso ao `WorkoutPlanRepository`.
